## 5. Decision log (chronological)

### 2026-04-25

- **Source data tarball abandoned.** `/home/sak185/data.tgz` (136 GB) was structurally inconsistent (`dicom_neg1` had extra `dicom_anon/` nesting; `dicom_neg5` absent; `dicom_neg4` 38 patients short). Pivoted to source from `/home/jjs374/DiaE` directly via `consolidate.py`.
- **Scratch base path corrected.** `/scratch/sak185/` is permission-denied on pioneer; the persistent user scratch is `/scratch/pioneer/users/<USER>/`. All paths in plan + scripts updated. See [reference_cwru_scratch.md](/home/sak185/.claude/projects/-home-sak185/memory/reference_cwru_scratch.md).
- **`Dicom upload/` subdir excluded.** 17 patients in `dicom/Dicom upload/` were exact byte-level duplicates of patients already in `dicom/`.
- **Layout flattened.** Pipeline now uses `positive/` and `negative/` instead of the original `dicom/` + `dicom_neg{1..5}/` batches. Cross-batch dedup happens at consolidation time, not at workplan build time.

### 2026-04-26

- **Mask-source mapping rewritten (v2.2).** Original `build_workplan.py` tried to assign one canonical mask per patient by parsing the existing nifti filename; this missed plain `ANONID.nii.gz` masks for 18 patients. New design: `build_workplan.py` emits one alignment_audit row per existing mask file; `realign_masks.py` shape-matches at runtime against ALL freshly-converted WATER outputs.
- **Multi-label masks accepted.** Some radiologist masks use values `{0,1,2}` or `{0,2}` — likely multi-lesion or multi-confidence annotations. The realign QC no longer fails on these; values are preserved as-is and flagged in `label_values` / `qc_notes`. Downstream training will need to decide on binarization vs multi-class.
- **5 patients hard-excluded.** ANON25C6C345BBDA, ANON474B6A632EC1, ANONB37185FC9DAF, ANONC0DC7E3FB015, ANONC4A3AEBA378D — only had masks on non-canonical (alt) WATER series, AND on visual inspection the lesion is not visible on the canonical sequence. Listed in `_common.py::EXCLUDED_PIDS`. Re-introduce by removing from the set.
- **Pilot QC passed.** User reviewed pilot outputs — orientation, intensity, mask overlay all look correct.
- **Phased rollout strategy adopted (v2.3).** Conversion split into Phase 1 (108 positives + 500 stratified negatives, ~15–30 min) and Phase 2 (remaining ~4,400 negatives, ~3–4h, on demand for SSL). Holdout: 22 pos + 100 neg. CV: 5-fold. Stratification: scanner model × slice-thickness binned (`≤4mm` vs `>4mm`) on canonical only. Seed: 42.

## 6. Known issues / sharp edges

- **Pilot output NOT in any split.** The 13 pilot + 23 investigation patients were converted before splits exist. Three options when Phase 1 runs: (a) re-convert in-place (idempotent), (b) leave them and assign splits to existing outputs, (c) wipe and start fresh. Recommend (a) — `convert_one_patient.py` is idempotent and overwrite is safe.
- **5 EXCLUDED_PIDS still have output dirs in /scratch.** Their `nifti_pos/<ANONID>/` dirs still exist from the pilot/investigation runs. `qc.py` should skip them (or we delete the dirs at Stage 3a as part of the split build). The exclusion is enforced at workplan build, so they won't be in `manifest.csv`, but the orphan files would remain. Cleanup TBD.
- **`dicom_neg5/_mapeamento_ids.csv`** mapping file (240 rows: original PatientID → ANONID) was excluded from consolidation. If we ever need to de-anonymize for a chart review, this CSV is the only key.
- **One rsync failure during consolidation:** ANON0C0B140ABF7F's source dir contained an orphan rsync tempfile (perms 600 owned by jjs374, permission denied). The 120 real DICOMs all copied successfully — the patient is intact. Only flagged here so future readers don't chase this.
- **`audit_mask_canonical.py` slice-axis bug history:** The first version compared mask `shape[2]` to expected n_dcm, but for these coronal LAVA volumes the slice axis is `shape[1]` (NOT `shape[2]` which is always 512). Fixed before publishing the audit. If you re-use this script, the fix is to take `min(shape, key=lambda d: abs(d - n_dcm))`.
