# Correctness Plan — Post‑Training Eval + Lesion‑Level Metrics (2026‑04‑29)

This plan covers correctness/integrity changes only (no efficiency work). It is scoped to the training/eval stack in `endo/` and is meant to be executable by a new agent with no prior context.

## 0) Clarification on CV split (answer to your question)

The code uses a classic 5‑fold CV split inside the cross‑validation cohort:

- For fold `f`, **validation = patients with `fold == f`**, **training = all other CV patients**.
- Holdout patients are always excluded (loaded only by `predict_holdout`).

Reference: `endo/data/manifest.py:44` (`fold_split`).

Therefore, when we say “other 4 folds,” we mean **the other folds’ validation partitions** (i.e., 4 disjoint val sets), not a train/val split inside each fold.

## 1) Requirements captured from the interview

### Must‑have features
1) **Final evaluation must use fresh inference** from a checkpoint (**best or last**), not deep‑eval caches.
2) **Threshold tuning:** for fold `f`, tune thresholds on **other folds’ validation sets**, then apply to fold `f` validation.
3) **Lesion‑level FROC** must use **actual lesion masks** (no proxy GT cuboid).
4) **Per‑call JSONL output:** for each predicted call and for each FN, output 3D volume (mm^3) and TP/FP/FN label.
5) **AUROC/AP** should be computed from **raw fused scores** (no size‑threshold filtering). Thresholding applies only to FROC/sensitivity.

### Explicit exclusions
- No holdout “touch guard” is needed.
- No deep‑eval pid guard is needed.
- No mmdet parity testing or related synthetic parity fixes.
- No efficiency/performance changes (handled by another agent).

## 2) High‑level design (what changes, conceptually)

### Current (simplified)
```
deep_eval cache (val slices) -> WBF -> threshold tuning on same data
                         -> final metrics
```

### New (target)
```
fresh inference from best/last ckpt
  -> WBF (raw)
  -> thresholds for fold f tuned on OTHER folds’ val sets
  -> per-fold metrics (AUROC/AP from raw; FROC/sens from thresholded)
  -> lesion-level FROC using GT masks
  -> per-call JSONL (TP/FP/FN + volumes)
```

Deep‑eval caches remain **training‑time only** (monitoring + HNM), not used for final CV.

## 3) Concrete changes by module (with file targets)

### 3.1 Evaluation flow — force fresh inference

**Files:** `endo/eval/run_eval.py`, `endo/config/eval.py`, `endo/cli/run_experiment.py`

**Change summary:**
- Replace deep‑eval cache loading with fresh inference from checkpoints.
- Support `eval_ckpt` in `EvalConfig` ("best" or "last").
- Ensure `predict_holdout` also uses `eval_ckpt`.

**Implementation steps:**
1) Add to `EvalConfig` (in `endo/config/eval.py`):
   - `eval_ckpt: Literal["best","last"] = "best"`
   - Optional: `emit_call_jsonl: bool = True`
2) Update `run_cv_evaluation` to:
   - Build a DataModule per fold (allow_holdout=False).
   - Load chosen ckpt (`fold{f}/ckpts/{best|last}.ckpt`) into `LesionDetectorLM`.
   - Run `inference_pass` over that fold’s **val pids**.
   - Store per‑fold **raw fused** predictions for later thresholding.
3) Update `run_holdout_inference` to use `eval_ckpt` rather than always `best.ckpt`.
4) Keep deep‑eval caches untouched (still generated by `PeriodicDeepEvalCallback` for training‑time monitoring).

### 3.2 Cross‑fold threshold tuning

**Files:** `endo/eval/run_eval.py`, `endo/eval/threshold_search.py`

**Goal:** thresholds for fold `f` are tuned on **other folds’ validation sets** only.

**Algorithm:**
```
for each fold f:
  tune_data = pooled raw predictions from folds != f (their val sets)
  thresholds_f = grid_search(tune_data)
  apply thresholds_f to fold f predictions

cv_pooled metrics:
  for each volume, use thresholds for its fold
```

**Notes:**
- This ensures no volume is evaluated with thresholds tuned on itself.
- `ensemble_threshold` for holdout can be defined as mean of per‑fold thresholds or via grid search on all folds (ok because holdout is independent). Plan: **mean of per‑fold thresholds** for transparency and stability.

### 3.3 AUROC/AP from raw fused scores only

**Files:** `endo/eval/run_eval.py`, `endo/eval/metrics.py`

**Change summary:**
- Compute AUROC/AP using **raw fused scores**.
- Compute FROC/sens@FP using **thresholded fused boxes**.

**Implementation approach:**
- Keep two prediction dicts per fold:
  - `raw_preds` (unfiltered WBF outputs)
  - `thr_preds` (after size‑threshold filtering)
- Use `raw_preds` to compute AUROC/AP.
- Use `thr_preds` to compute FROC/sens@FP.
- Merge into one metrics dict before writing CSV.

### 3.4 Lesion‑level FROC using GT masks

**Files:** `endo/eval/metrics.py`, `endo/eval/froc.py`, `endo/eval/run_eval.py`

**Change summary:**
- Pass **real lesion masks** to `compute_froc` instead of proxy cuboid.
- Ensure masks are aligned to the detection map’s `(Y, Z, X)` shape.

**Implementation details:**
- Load lesion masks from cache for each pid:
  - cache arrays are `(X, Y, Z)` → transpose to `(Y, Z, X)`.
- For negatives, use all‑zero masks.
- Update `compute_volume_metrics` to accept optional `gt_masks` and pass through to `compute_froc`.
- Leave `froc.py` fallback proxy for cases when masks are not provided (unit tests).

### 3.5 Per‑call JSONL output (TP/FP/FN + volume)

**Files:** new `endo/eval/calls.py`, plus wiring in `endo/eval/run_eval.py`.

**Definition of a call:** 3D connected component from the detection map.

**Call extraction steps:**
1) Build detection map `(Y,Z,X)` from **raw fused boxes** (same as `froc.py`).
2) Extract 3D connected components (use 26‑connectivity; same as lesion masks).
3) For each component (pred call):
   - `call_score = max(voxel scores in component)`
   - `call_volume_mm3 = n_voxels * spacing_mm^3`
   - `bbox` and `centroid` in `(Y,Z,X)`
4) Assign TP/FP using **one‑to‑one greedy** match to GT lesions:
   - A call matches a GT lesion if call centroid is inside GT lesion mask.
   - For each GT lesion, only the highest‑score matching call is TP; others FP.
5) FNs = any GT lesion with no matched call.

**Pass flag:** include `passes_threshold` based on tuned thresholds and call size:
- Compute `max_dim_mm` from call’s XZ bounding box.
- If `max_dim_mm >= split`, compare score to `large_thr`; else `small_thr`.

**JSONL schema (per line):**
```
{
  "run_id": str,
  "entrypoint": "cv"|"holdout",
  "fold": int|null,
  "patient_id": str,
  "call_id": str,               # unique per run
  "call_type": "tp"|"fp"|"fn",
  "score": float|null,          # null for FN
  "passes_threshold": bool|null,
  "volume_mm3": float,
  "voxel_count": int,
  "bbox_yz_x": [y0,y1,z0,z1,x0,x1],
  "centroid_yz_x": [y,z,x],
  "gt_lesion_id": str|null      # for TP/FN
}
```

**Output paths:**
- CV: `runs/<exp>/eval/per_call_<run_id>.jsonl`
- Holdout: `runs/<exp>/holdout/run_<ts>_<uuid>/per_call_<run_id>.jsonl`

### 3.6 GRU rescoring with fresh eval

**Files:** `endo/gru/feature_cache.py`, `endo/gru/rescorer.py`, `endo/eval/run_eval.py`

**Change summary:**
- When `--use-gru` in eval, rebuild feature cache from the chosen ckpt before rescoring.

**Implementation approach:**
1) Add helper in `feature_cache.py`: `extract_features_for_pids(ckpt_path, pids, output_dir, batch_size)`.
2) In `run_eval.py`, before rescoring fold `f`:
   - Build a temp feature cache dir under `runs/<exp>/eval/feature_cache/fold{f}/`.
   - Extract features for that fold’s val pids using the **same ckpt** used for fresh inference.
   - Rescore using that cache (do not reuse stale caches).

## 4) Detailed step‑by‑step implementation checklist

1) **Config updates**
   - `endo/config/eval.py`: add `eval_ckpt`, `emit_call_jsonl`.
   - Update `experiments/*.py` defaults if needed (or let eval use default). No CLI overrides.

2) **Fresh inference path**
   - In `endo/eval/run_eval.py`, replace `_latest_deep_eval_npz` usage with fresh inference.
   - New helper: `_load_ckpt_for_fold(fold, ckpt_choice)` returning `LesionDetectorLM` on device with EMA applied.
   - Build a `LesionDataModule` for each fold (allow_holdout=False) and run `inference_pass` on val pids.
   - Store per‑fold `slice_scores` and `raw_preds`.

3) **Cross‑fold threshold tuning**
   - Compute `thresholds_by_fold[f]` using pooled raw preds from folds != f.
   - Apply these to fold f predictions for FROC/sens.
   - Define `ensemble_threshold` as mean of per‑fold thresholds for holdout use.

4) **Metric computation split (raw vs thresholded)**
   - `AUROC/AP`: from raw fused `score`.
   - `FROC/sens@FP`: from thresholded boxes.
   - Merge into a single metrics dict before CSV writing.

5) **Lesion masks for FROC**
   - Load lesion masks from cache for each pid; transpose to `(Y,Z,X)`.
   - Pass into `compute_volume_metrics(..., gt_masks=...)`.
   - Update `compute_volume_metrics` and `compute_froc` signatures accordingly.

6) **Per‑call JSONL**
   - New module `endo/eval/calls.py` with:

     - `extract_pred_calls` (connected components)
     - `match_calls_to_gt` (greedy matching)
     - `write_calls_jsonl`
   - Wire into `run_cv_evaluation` and `run_holdout_inference`.

7) **GRU with fresh eval**
