# `data/` — provenance, layout, and rules for future agents

This directory holds the **Phase 1 development cohort** for the diaphragmatic
endometriosis object-detection project: 608 patient MR volumes, 108 ground-truth
lesion masks, derived liver segmentations and ROIs, and the metadata files that
glue them together.

The data here is **not committed to git** (the NIfTI volumes alone are ~19 GB).
Only `splits.json`, `patient_id_mapping.csv`, `README.md`, and this `CLAUDE.md`
are tracked. Everything else is regenerated by re-running the pipeline in
`scripts/` against the upstream consolidated source on /scratch.

---

## 1. Upstream story (how this folder came to exist)

1. **Source delivery.** Anonymised DICOMs landed at `/home/jjs374/DiaE/`:
   one `dicom/` batch (positives) + five `dicom_neg{1..5}` batches (negatives) +
   prior radiologist-converted NIfTIs and lesion masks under `nifti/` and
   `masks/`. Total ~5,122 patient directories.

2. **Consolidation** (`scripts/consolidate.py`). Deduped across negative batches
   (same ANONID in multiple batches → keep largest), removed pos/neg ANONID
   overlaps (positive wins), flattened to
   `/scratch/pioneer/users/sak185/dia-endo-conversion/input/{positive,negative,nifti,masks}/`.

3. **Prescan + workplan** (`scripts/prescan.py`, `scripts/build_workplan.py`).
   Indexed every DICOM series and selected the **canonical sequence** per
   patient: `WATER` from a `DERIVED\PRIMARY\DIXON\WATER\MAGNITUDE` Dixon LAVA
   coronal acquisition (the "WATER: COR LAVA DIAF." series description). FAT,
   in-phase, out-phase, and alternate-water series were tracked but excluded
   from the supervised cohort.

4. **DICOM → NIfTI conversion** (`scripts/convert_one_patient.py` driven by
   `slurm/convert_phase{1,2}.slurm`). dcm2niix v1.0.20250505. Two SLURM array
   phases:
   - **Phase 1** = the 608-patient supervised dev cohort (this folder).
   - **Phase 2** = the remaining ~4,476 negatives, intended as an SSL pretraining
     pool. **Phase 2 was not migrated to /home** and stays on /scratch; it
     appears in `manifest.csv` with `transferred_to_home=False` and
     `split=phase2_unsupervised`.

5. **QC + mask realignment** (`scripts/qc.py`, `scripts/realign_masks.py`,
   `scripts/audit_mask_canonical.py`). 126 positive masks audited and
   realigned to their canonical volume; all `qc_status=ok`.

6. **Stratified splits** (`scripts/build_splits.py`). Frozen 5-fold CV +
   122-patient holdout, **seed=42**:
   - Positives stratified by `manufacturer_model_name` only (thickness collapsed
     because Explorer-thick has only 6 patients and Artist-thick 0).
   - Negatives stratified by `manufacturer_model_name × slice_thickness_bin`
     (≤4 mm vs >4 mm) on the canonical sequence.
   - Largest-remainder rounding for proportional allocation.
   - Output: `splits.json` is authoritative; `manifest.csv["split"]` carries the
     same assignment per row.

7. **Mnemonic renaming + migration to /home**
   (`scripts/generate_patient_names.py` → `scripts/migrate_to_home.py`).
   Each `ANON…` ID was mapped to a deterministic 3-word underscore mnemonic
   (`amber_bear_quartz` style) drawn from `scripts/wordlists.json`. The 608
   Phase 1 patients were copied here with mnemonic filenames. dcm2niix BIDS
   sidecars were consolidated into a single `sidecars.jsonl`. README.md was
   auto-generated. **The migration is idempotent** (sha256 match on NIfTIs).

8. **Liver segmentation** (`scripts/run_totalseg.py`). TotalSegmentator
   `task=total_mr roi_subset=liver`, multilabel, full-res normal mode, on GPU,
   6–8 workers. 608/608 succeeded. Wrote binary uint8 masks to
   `liver_masks/<bucket>/<cohort>/<mnemonic>_liver_mask.nii.gz` and back-filled
   `liver_mask_path / liver_mask_sha256 / liver_voxel_count` columns in
   `manifest.csv`.

9. **Liver ROI dilation** (`scripts/dilate_segmentations.py`). 20 mm
   physical-space dilation of each liver mask via anisotropic distance
   transform, using each NIfTI's own voxel sizes (do NOT assume a uniform
   grid across the cohort). Output to `liver_rois/...` with
   `liver_roi_path / liver_roi_margin_mm` back-filled into `manifest.csv`.

Full design rationale and decision history lives in `agent/`:
`migration-plan.md`, `totalseg-plan.md`, `convert-plan-v2.md`, and the two
`handoff-2026-04-26*.md` snapshots.

## 2. Layout

```
data/
├── raw/                            ← NIfTI volumes (608 files, ~19 GB)
│   ├── holdout/
│   │   ├── positive/<mnemonic>.nii.gz                    (22)
│   │   └── negative/<mnemonic>.nii.gz                   (100)
│   └── cross-validation/
│       ├── positive/<mnemonic>.nii.gz                    (86)
│       └── negative/<mnemonic>.nii.gz                   (400)
│
├── lesion_masks/                   ← radiologist GT, positives only (108 files)
│   ├── holdout/positive/<mnemonic>_mask.nii.gz           (22)
│   └── cross-validation/positive/<mnemonic>_mask.nii.gz  (86)
│
├── liver_masks/                    ← TotalSegmentator binary masks (608 files)
│   ├── holdout/{positive,negative}/<mnemonic>_liver_mask.nii.gz
│   └── cross-validation/{positive,negative}/<mnemonic>_liver_mask.nii.gz
│
├── liver_rois/                     ← liver_masks dilated by 20 mm (608 files)
│   ├── holdout/{positive,negative}/<mnemonic>_liver_roi.nii.gz
│   └── cross-validation/{positive,negative}/<mnemonic>_liver_roi.nii.gz
│
├── _pipeline/                      ← runtime artifacts (logs, timing, VRAM)
│
├── manifest.csv          5,060 rows; transferred_to_home gates physical presence
├── sidecars.jsonl        608 lines, one BIDS sidecar + provenance per patient
├── splits.json           seed=42 fold assignments; authoritative for splits
├── patient_id_mapping.csv   anon_id ↔ mnemonic_id, immutable
├── README.md             auto-generated migration overview (slightly stale*)
└── CLAUDE.md             this file
```

\* `README.md` shows raw filenames as `<mnemonic>.nii.gz` for masks too; the
actual on-disk convention is `<mnemonic>_mask.nii.gz`,
`<mnemonic>_liver_mask.nii.gz`, `<mnemonic>_liver_roi.nii.gz`. **Trust the
filesystem and the `*_path` columns in `manifest.csv`, not README's globs.**

### Bucket × cohort × split

| bucket | cohort | total | per-fold (CV) |
|---|---|---:|---|
| holdout | positive | 22 | n/a (locked test) |
| holdout | negative | 100 | n/a |
| cross-validation | positive | 86 | fold0:18, fold1:18, fold2:17, fold3:17, fold4:16 |
| cross-validation | negative | 400 | fold0:82, fold1:81, fold2:79, fold3:79, fold4:79 |

**Two physical buckets, five logical folds.** All 486 CV patients live in one
`cross-validation/` subtree; the per-patient fold ID is in
`manifest.csv["split"]` and `splits.json["assignments"]`. Training code reads
one of these at runtime — **do not encode fold IDs in directory paths.**

## 3. Key files in detail

### `manifest.csv` (5,060 rows)

Tracks **every** patient ever seen by the pipeline, not just the 608 in this
folder. Filter `transferred_to_home == True` to scope to what's physically
here.

Useful columns:
- `mnemonic_id`, `anon_id`, `bucket`, `cohort`, `split` (fold0..fold4 / holdout
  / phase2_unsupervised), `soft_negative` (true for the 57 reclassified ex-
  positives that got demoted to negatives during workplan v2.3).
- `raw_path`, `lesion_mask_path`, `liver_mask_path`, `liver_roi_path` —
  relative to `data/`.
- `sha256_raw`, `liver_mask_sha256` — content hashes for idempotent re-runs.
- `liver_voxel_count`, `liver_roi_margin_mm` — sanity-check derivatives.
- Acquisition geometry: `slice_thickness_mm`, `pixel_spacing_x_mm`, `shape`,
  `n_slices_actual`, `image_type`, `series_description`, `scanner_model`,
  `magnetic_field_strength`.

### `sidecars.jsonl` (608 lines)

One JSON object per transferred patient: `mnemonic_id`, `anon_id`, `bucket`,
`split`, `cohort`, `raw_path`, and a complete dcm2niix BIDS `sidecar` block
(Modality, ImageType, EchoTime, RepetitionTime, ImageOrientationPatientDICOM,
ConversionSoftware, etc.).

Read with `polars.read_ndjson("data/sidecars.jsonl")` or `jq -c '.'`.

### `splits.json`

```json
{
  "seed": 42,
  "n_folds": 5,
  "stratification_keys": { ... },
  "phase1_targets": { "holdout_pos": 22, "holdout_neg": 100, "cv_pos": 86, "cv_neg": 400, "phase1_total": 608 },
  "pool_sizes": { "true_positives": 108, "true_negatives": 4890, "soft_negatives": 57, "fat_only_no_canonical": 29 },
  "soft_negative_pids": [...],
  "assignments": { "<anon_id>": "fold0" | "holdout" | ... }
}
```

`assignments` is the authoritative source for fold ID. **Re-running
`scripts/build_splits.py` with seed=42 must reproduce the same file** —
splits are frozen.

### `patient_id_mapping.csv`

`anon_id,mnemonic_id` — deterministic, immutable. Generated by
`scripts/generate_patient_names.py` against `scripts/wordlists.json`. If you
ever regenerate, you will rename every file in this tree; don't, unless you
also re-run `migrate_to_home.py`.

### `_pipeline/`

Runtime byproducts of `run_totalseg.py` and friends:
`pipeline_run*.log`, `timing_full.csv`, `timing_test.csv`, `vram_log_test.csv`,
`test12_patients.csv`. Useful for diagnosing slow patients or VRAM regressions;
**not committed**, fully regenerable.

## 4. Conventions and gotchas

- **MR characteristics.** All 608 volumes are GE 1.5 T, 3D Dixon LAVA coronal
  WATER images — `MRAcquisitionType=3D`, axial-view shape typically
  `512×NSlice×512` with NSlice ∈ ~98–160, slice thickness ~1.7–4 mm. Patient
  position FFS. Body part PELVIS or ABDOMEN. **No CT.** When picking
  preprocessing (intensity normalization, augmentations), reach for what works
  for MR, not CT — TotalSegmentator was deliberately invoked with
  `task=total_mr`.

- **Voxel grid is not uniform.** Each NIfTI carries its own affine and zooms.
  Code that does physical-space operations (dilation, distance, resampling)
  must read `nib.load(...).header.get_zooms()` per file, not assume a fixed
  spacing. `dilate_segmentations.py` is the worked example.

- **Holdout is locked.** Don't read `data/raw/holdout/` or `data/lesion_masks/holdout/`
  during training, hyperparameter selection, model picking, or any iterative
  loop. The bucket separation is physical so a glob mistake is hard to make,
  but it's still a mistake to undo.

- **`soft_negative=True`** marks 57 patients who lacked a usable mask file at
  workplan time and were reclassified from positive→negative. They live in the
  `negative` cohort and contribute to negative folds. Treat them as negatives
  for training, but if you ever do error analysis on negatives, check this
  column first — they may be the most clinically interesting "negatives" in the
  set.

- **`phase2_unsupervised` patients are not on disk here.** Filter by
  `transferred_to_home == True` (or equivalently
  `split.is_in(["holdout", "fold0", ..., "fold4"])`) before any I/O.

- **Mnemonic IDs are case-stable, underscore-separated, three words.**
  `amber_bear_quartz` is one patient. They are deterministic given the
  wordlist and seed, but **not human-meaningful** — don't read clinical
  signal into the words.

- **Lesion masks exist only for positives** (108 files, not 608). Object
  detection training samples need to handle the missing-label case for
  negatives explicitly (empty box list, not a zero-tensor mask).

- **Filenames have suffixes** (`_mask`, `_liver_mask`, `_liver_roi`); the
  `<mnemonic>` is the join key. Use `manifest.csv` paths or
  `Path(...).name.split("_")[:3]` to recover the mnemonic.

## 5. Reproducing this folder from upstream

If you need to rebuild from scratch, the commands roughly are (see
`agent/migration-plan.md` and `agent/totalseg-plan.md` for the actual
invocations):

```bash
# 0. Source assumed at /home/jjs374/DiaE
uv run -m scripts.consolidate           --output-root /scratch/.../input
uv run -m scripts.prescan               --output-root /scratch/.../input
uv run -m scripts.build_workplan        --workplan /scratch/.../workplan
uv run -m scripts.convert_one_patient   # via slurm/convert_phase{1,2}.slurm
uv run -m scripts.qc
uv run -m scripts.realign_masks
uv run -m scripts.build_splits          # seed=42 → splits.json
uv run -m scripts.generate_patient_names --wordlists scripts/wordlists.json
uv run -m scripts.migrate_to_home --execute
uv run -m scripts.run_totalseg --data-root data --execute
uv run -m scripts.dilate_segmentations --input-dir data/liver_masks --output-dir data/liver_rois --execute
```

Anything in this directory that is *not* the four committed files (this
`CLAUDE.md`, `README.md`, `splits.json`, `patient_id_mapping.csv`) is a
deterministic function of the upstream source plus those four files. Don't
treat it as ground truth — `scripts/` is the ground truth.
